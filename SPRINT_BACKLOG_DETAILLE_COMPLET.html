📖 Comment lire ce document ?

Ce document présente toutes les fonctionnalités du système de gestion de quiz. Chaque Epic représente une grande fonctionnalité. Chaque User Story (US) décrit ce qu'un utilisateur peut faire. Chaque Tâche détaille le travail technique à réaliser.

🎯 EPIC 1 : GESTION ADMINISTRATIVE DES QUIZ (29 heures)

🎯 Objectif de cet Epic

Permettre aux administrateurs de créer, modifier, supprimer et visualiser les quiz. C'est la partie "backoffice" où les enseignants gèrent le contenu des évaluations.

US-1.1 En tant qu'administrateur, je veux créer un nouveau quiz afin de proposer des évaluations aux étudiants

💡 Explication de cette User Story

Cette fonctionnalité permet à un enseignant de créer un nouveau quiz avec un titre, une description, une durée maximale, un seuil de réussite (par exemple 50%), et un nombre maximum de tentatives. Le quiz doit être associé à un chapitre de cours.

T1.1.1 ⏱️ 30 min
Tâche : En tant que concepteur, je dois préparer le diagramme de classe de conception pour l'entité Quiz

📝 Détails de la tâche

Créer un schéma qui montre la structure de la table Quiz dans la base de données. Ce schéma inclut toutes les colonnes : id, titre, description, état, durée maximale, seuil de réussite, nombre maximum de tentatives, et la relation avec le chapitre.

👨‍💻 Équipe Backend
T1.1.2 ⏱️ 1h
Tâche : En tant qu'Admin BD, je dois créer la table quiz avec les colonnes: id, titre, description, etat, duree_max_minutes, seuil_reussite, max_tentatives, chapitre_id

📝 Détails de la tâche

Créer physiquement la table dans la base de données MySQL/PostgreSQL avec toutes les colonnes nécessaires. Définir les types de données (VARCHAR pour le titre, INT pour la durée, etc.) et les contraintes (NOT NULL pour les champs obligatoires, clé étrangère vers la table chapitre).

👨‍💻 Équipe Backend
T1.1.3 ⏱️ 1h
Tâche : En tant que développeur, je dois créer l'entité Quiz.php avec toutes les propriétés et annotations de validation

📝 Détails de la tâche

Créer la classe PHP Quiz.php dans Symfony qui représente la table quiz. Ajouter toutes les propriétés (titre, description, état, etc.) avec les annotations Doctrine pour la base de données et les annotations de validation (NotBlank, Length, Range) pour vérifier que les données sont correctes avant l'enregistrement.

👨‍💻 Équipe Backend
T1.1.4 ⏱️ 1h
Tâche : En tant que développeur, je dois créer le formulaire QuizType avec EntityType pour la relation chapitre

📝 Détails de la tâche

Créer le formulaire Symfony qui permet de saisir les informations du quiz. Le formulaire contient des champs texte pour le titre et la description, un menu déroulant pour sélectionner le chapitre, des champs numériques pour la durée et le seuil de réussite. Utiliser EntityType pour afficher la liste des chapitres disponibles.

👨‍💻 Équipe Backend
T1.1.5 ⏱️ 45 min
Tâche : En tant que développeur, je dois créer l'action new() dans QuizController pour gérer la création

📝 Détails de la tâche

Créer la méthode new() dans le contrôleur QuizController qui affiche le formulaire de création. Quand l'utilisateur soumet le formulaire, cette méthode valide les données, enregistre le nouveau quiz dans la base de données, et affiche un message de confirmation. En cas d'erreur, elle réaffiche le formulaire avec les messages d'erreur.

👨‍💻 Équipe Backend
T1.1.6 ⏱️ 2h
Tâche : En tant que développeur frontend, je dois créer le template new.html.twig avec design glass morphism

📝 Détails de la tâche

Créer la page HTML/Twig qui affiche le formulaire de création de quiz. Appliquer un design moderne "glass morphism" (effet de verre transparent avec flou). Styliser les champs du formulaire avec des couleurs attrayantes, des bordures arrondies, et des effets au survol. Rendre le formulaire responsive pour mobile et tablette.

🎨 Équipe Frontend
T1.1.7 ⏱️ 1h
Tâche : En tant que développeur, je dois implémenter la validation métier dans QuizManagementService

📝 Détails de la tâche

Créer un service QuizManagementService qui contient les règles métier. Par exemple : vérifier qu'un quiz a bien un chapitre associé, que le seuil de réussite est entre 0% et 100%, que la durée est positive. Ce service centralise toute la logique de validation pour éviter de dupliquer le code dans les contrôleurs.

👨‍💻 Équipe Backend
T1.1.8 ⏱️ 1h
Tâche : En tant que testeur, je dois préparer les cas de test pour la création de quiz

📝 Détails de la tâche

Préparer un document qui liste tous les scénarios de test à vérifier : création réussie avec données valides, erreur si le titre est vide, erreur si aucun chapitre n'est sélectionné, erreur si le seuil de réussite est négatif, etc. Définir les données de test et les résultats attendus pour chaque scénario.

✅ Équipe QA
T1.1.9 ⏱️ 45 min
Tâche : En tant que testeur, je dois exécuter les tests de validation (champs obligatoires, formats, contraintes)

📝 Détails de la tâche

Exécuter manuellement tous les cas de test préparés précédemment. Vérifier que le système affiche bien les messages d'erreur appropriés quand les données sont invalides, et que le quiz est créé correctement quand les données sont valides. Documenter les résultats et signaler les bugs éventuels.

🔗 EPIC 5 : RELATION QUIZ-CHAPITRE OBLIGATOIRE (3 heures)

🔒 EPIC 7 : SÉCURITÉ DU SYSTÈME (8 heures)

🎯 Objectif de cet Epic

Sécuriser l'application contre les attaques courantes (CSRF, XSS, injection SQL) et contrôler l'accès aux fonctionnalités selon les rôles utilisateurs (ROLE_ADMIN pour le backoffice, ROLE_ETUDIANT pour le frontoffice).

US-7.1 En tant que système, je veux protéger les formulaires contre les attaques
T7.1.1 ⏱️ 30 min
Tâche : Activer la protection CSRF dans security.yaml

📝 Détails

CSRF (Cross-Site Request Forgery) : attaque où un site malveillant force l'utilisateur à effectuer des actions non désirées. La protection CSRF ajoute un token unique à chaque formulaire qui doit être validé lors de la soumission.

👨‍💻 Équipe Backend
T7.1.2 ⏱️ 2h
Tâche : Implémenter la validation serveur avec Symfony Validator
👨‍💻 Équipe Backend
T7.1.3 ⏱️ 1h
Tâche : Sécuriser l'affichage avec Twig auto-escape

📝 Détails

Auto-escape protège contre les attaques XSS (Cross-Site Scripting) en échappant automatiquement les caractères HTML spéciaux. Par exemple, <script> devient &lt;script&gt; et ne s'exécute pas.
