🚨 URGENT - INSTALLATION REQUISE APRÈS PULL
============================================

Salut l'équipe! 👋

Si la météo et les emails ne fonctionnent pas, c'est normal!

⚠️ VOUS DEVEZ INSTALLER LES PACKAGES!

Le dossier vendor/ n'est PAS dans Git, vous devez l'installer.

📦 INSTALLATION EN 3 ÉTAPES (5 minutes)
========================================

1️⃣ Installer les packages:
   
   composer install

   (Durée: 2-5 minutes)

2️⃣ Créer .env.local:
   
   copy .env.local.example .env.local

3️⃣ Éditer .env.local et coller:

###> symfony/mailer - SENDGRID ###
MAILER_DSN=sendgrid+api://SG.VOTRE_CLE_SENDGRID_ICI@default
###< symfony/mailer ###

###> openweathermap ###
WEATHER_API_KEY=VOTRE_CLE_OPENWEATHERMAP_ICI
###< openweathermap ###

4️⃣ Redémarrer le serveur:
   
   symfony server:start

✅ TESTER
=========

- Météo: http://localhost:8000/events
- Email: Participez à un événement

📖 GUIDE COMPLET
================

Si problèmes, voir: INSTALLATION_PACKAGES_EQUIPE.md

🔧 DÉPANNAGE RAPIDE
===================

❌ "composer: command not found"
   → Installez Composer: https://getcomposer.org/

❌ "Class not found"
   → Vous avez oublié: composer install

❌ Météo ne s'affiche pas
   → Vérifiez .env.local existe et contient les clés

❌ Emails non envoyés
   → Vérifiez .env.local et redémarrez serveur

⚠️ NE PAS OUBLIER
=================

- NE JAMAIS commiter .env.local sur Git
- NE JAMAIS commiter vendor/ sur Git
- Toujours faire composer install après un pull
